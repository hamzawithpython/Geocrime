About GeoCrime

GeoCrime is a crime-aware route planning web app. It analyzes historical crime data and uses a custom A* routing engine to suggest paths that balance safety and travel time.

Built as a Final Year Project (BSc Hons Computer Science), GeoCrime aims to make urban travel more informed and confident.

Try the Map

Mission

Traditional navigation optimizes for distance or time and rarely considers risk exposure. GeoCrime integrates crime history into routing to help users avoid high-risk areas, visualize nearby incidents, and make better decisions on the go.

How It Works

1) Data Ingestion

Crime data (historical records by location & type) is loaded (CSV/JSON; migrating to PostgreSQL).

2) Crime-Weighted Graph

Using OSMnx + NetworkX, we build a road graph. Edges near higher crime density receive higher costs.

3) Safer Routing

A custom A* algorithm finds a path that balances safety and efficiency, then computes a Safety Count & Danger Score.

Key Features

Crime-Aware Routing

Custom A* with crime-weighted edges.

Interactive Map

Pick start/destination, view routes, and toggle layers.

Risk Labels

Safety Count & Danger Score → Safe / Moderate / High Risk.

Layers & Visuals

Markers, heatmaps, and density overlays.

City-Agnostic

Designed for Chicago first, portable to other cities.

Dev-Friendly

Django backend, Google Maps front-end, clear API endpoints.

Technology Stack

  • Frontend: Google Maps JS API, Tailwind CSS, Flowbite
  • Backend: Django (Python 3.12)
  • Routing: OSMnx + NetworkX (crime-weighted A*)
  • Data: CSV/JSON → PostgreSQL (in progress)
  • Utilities: Geopy, Ngrok for remote testing

Ethics & Privacy

GeoCrime uses historical, publicly sourced crime data where available. We avoid collecting personal data from users, and we clearly label risk estimates as advisory, not definitive. Users should still exercise judgment and follow local guidance.

Roadmap
